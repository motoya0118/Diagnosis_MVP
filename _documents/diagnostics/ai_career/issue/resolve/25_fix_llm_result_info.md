## 現状の問題点
以下の画面でローカルストレージに登録されるllm_resultの情報が過剰すぎてハッキングの手助けになってしまう。
- _documents/diagnostics/ai_career/front/03_共通診断画面.md
- _documents/diagnostics/ai_career/front/04_診断結果画面.md

## 解決方針
ローカルストレージに格納するのは以下の情報のみでよく、それ以外の情報は除外する。
※現状、モデル情報、usage情報等も含まれてしまっている
```json
{
  1:{
    "name": <outcome_table.name>,
    "total_match":{
      "score": <0~100 職種への総合マッチ度>,
      "reason": <スコアの根拠>,
    },
    "personality_match":{
      "score": <0~100 職種への性格マッチ度>,
      "reason": <スコアの根拠>
    },
    "work_match":{
      "score": <0~100 業務系、開発系の区分のマッチ度>,
      "reason": <スコアの根拠>
    }
  },
  2:{...},
  3:{...}
}
```

- [x] フロントエンドの型定義・インターフェース（TypeScript 等）がある場合は、新しい保存フォーマットに合わせて更新し、不要なプロパティの参照が残っていないか確認する。
