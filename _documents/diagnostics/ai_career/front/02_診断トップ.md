# 2. 診断トップ画面設計

## ページ概要
- Path: `/diagnostics/ai_career`
- 目的: 診断説明・スタート導線の提供、および関連コンテンツへのナビゲーション。

## コンテンツ構成
- ヒーローセクション: タイトル、サブコピー、代表イラスト（任意）。
- 診断概要: 所要時間、質問数、結果の出力形式、個人情報取り扱いの注釈。
- `診断を開始` ボタン: 共通診断画面への遷移リンク。
- FAQ/補足情報（任意）: 診断の活用方法、推奨チームなど。

## 画面遷移
- ボタン押下で `/diagnostics/common_qa?diagnostic_code=ai_career` へ遷移。
  - URLパラメータで `diagnostic_code` を必ず付与する。
  - ログイン状態に依存せず遷移可。

## 状態/データ取得
- 静的コンテンツ中心のため API コール不要。
- 将来的に診断メタをAPI化する場合に備え、`diagnostic_code` に紐づくメタ情報を `SWR` 等で取得可能な構造に留める。

## UI要件
- ボタンは主要CTAとして視認性を確保。
- モバイル表示ではスタックレイアウトで折り返す。
- `診断を開始` 押下時には軽いフェードアウトなどで遷移を演出（ユーザー体験向上）。
